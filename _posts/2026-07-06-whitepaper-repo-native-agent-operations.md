We've published the studio's founding whitepaper —
[**Repo-Native Agent Operations**](/whitepaper/), v1.0.0.

It argues that the delivery gap in agentic software is a *system-of-record* gap,
not a tooling gap, and describes six architectural commitments that make a
one-operator studio's every claim checkable by `git log`: the gnome contract,
propose-don't-decide, the two-layer output contract, trust-tier manifests,
runner-level cost receipts, and the guidance ledger. It evaluates them against
this studio's own bootstrap and pilot — the numbers all trace to these repos.

The paper is versioned content: the canonical URL
[`/whitepaper/`](/whitepaper/) always serves the latest release, with a changelog
in the paper itself. This is v1.0.0, ratified 2026-07-06.
