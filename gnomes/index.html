A gnome is a single agent that handles one task end to end. Platform gnomes serve the studio itself; project gnomes serve one or more projects. Gnomes work in knolls — teams with a shared purpose and a shared knowledge base that every member carries into its runs. All of them live in the same registry, honor the same contract, and leave the same receipts.

{%- comment -%} Gnome-igotchi (EPIC3-09, platform ADR-0036): the display-only personality overlay — derived from real run history, behind the CI-enforced firewall (personality artifacts are never loadable by the harness) — rides in on each gnome's `igotchi` block (merged by scripts/build_data.py from igotchi.json on the platform ledger branch) and renders inside the theme's gnome-card. Every trait is traceable: rule + evidence live in _data/studio/igotchi.json. {%- endcomment -%} {%- for k in knolls -%} {%- assign members = gnomes | where: "knoll", k.name -%} {%- if members.size > 0 -%}
